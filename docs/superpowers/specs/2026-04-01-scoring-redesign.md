# 점수 체계 재설계 — 미래가치 + 기술적 전환 통합 스코어

**날짜**: 2026-04-01  
**상태**: 설계 확정, 구현 대기

---

## 배경 및 목표

### 문제

현재 시스템은 SMS 신호(라씨/스톡봇/퀀트) 기반으로 설계되어 다음 구조적 한계가 있다:

- **신호 없는 종목 = 평가 불가**: 삼성전자, 한국전력처럼 신호가 드문 대형 가치주가 D등급
- **대형주 52주 저점 = 위험 신호**: 실제로는 최적 진입 구간인데 2점 부여
- **수급 기준 불합리**: 시총 대비 비율로 계산해 대형주는 수천억 매수도 3점
- **신호 11일+ 경과 = 0점**: 가치 지표와 무관하게 점수 소멸

### 목표

> **어떤 종목이든 저평가되어 있고 기술적 전환점에 있으면 높은 점수를 받는다.**

- 전 종목(~4,242개) 동일 기준 평가
- SMS 신호는 보조 가산점, 없어도 A등급 가능
- 대형/소형주 동일 기준 적용 (단, 티어별 가중치는 차등 유지)
- 점수 체계의 유효성을 백테스트로 검증 가능

## 점수 구조

### 종합점수 공식

```
종합점수 = (기술전환 × W1 + 수급강도 × W2 + 가치매력 × W3 + 신호보너스 × W4)
         × (1 - 리스크페널티)
```

모든 축은 **0~100 정규화** 후 가중 합산.

### 티어별 가중치

| | 기술전환(W1) | 수급강도(W2) | 가치매력(W3) | 신호보너스(W4) |
|---|---|---|---|---|
| **대형주** (시총 1조+) | 30% | 25% | 35% | 10% |
| **중형주** (1천억~1조) | 35% | 25% | 30% | 10% |
| **소형주** (1천억 미만) | 38% | 28% | 24% | 10% |

### 등급 기준 (현행 유지)

| 등급 | 점수 | 의미 |
|---|---|---|
| A+ | 90+ | 적극매수 |
| A  | 80+ | 매수 |
| B+ | 65+ | 관심 |
| B  | 50+ | 보통 |
| C  | 35+ | 관망 |
| D  |  0+ | 주의 |

## 축 1. 기술적 전환 (Technical Reversal) — 0~100

**원칙**: 대형·소형 동일 기준. "전환점"에 있는 종목에 최고점.  
**데이터**: `daily_prices` (최소 20일, 권장 60일)  
**최대 원점수**: 65점 → 0~100 정규화

### 가점 항목

| 조건 | 원점수 | 설명 |
|---|---|---|
| MA5가 MA20 상향 돌파 (1~5일 내) | +25 | 골든크로스 초입 — 최고 전환 신호 |
| RSI 25~45 구간 (과매도 회복 중) | +20 | 매수 적정 구간 |
| 52주 저점 +5~20% 반등 중 | +20 | 바닥 확인 후 진입 구간 |
| 거래량 20일 평균 1.5배+ (당일) | +15 | 상승 확인 거래량 |
| 볼린저 하단 터치 후 양봉 반등 | +15 | 기술적 지지 반등 |
| MA5 > MA20 > MA60 완전 정배열 | +10 | 상승 추세 유지 확인 |
| 52주 고점 90%+ 구간 | +10 | 강한 모멘텀 지속 |
| 연속 하락 후 첫 양봉 (3일+ 하락 후) | +10 | 단기 반등 시작 |

### 감점 항목

| 조건 | 원점수 | 설명 |
|---|---|---|
| RSI 70+ (과매수) | -15 | 단기 과열 |
| 5일 누적 -15% 이상 급락 | -10 | 낙폭 과대 위험 |
| MA5 < MA20 < MA60 역배열 심화 | -5 | 하락 추세 지속 |

**데이터 부족 (20일 미만)**: 기술전환 0점, 다른 축은 정상 평가.

## 축 2. 수급 강도 (Supply Strength) — 0~100

**원칙**: 시총 비율 폐기. **방향 전환 + 연속성**이 핵심.  
**데이터**: `stock_cache` (foreign_net_qty, institution_net_qty, foreign_streak, institution_streak, foreign_net_5d, institution_net_5d, short_sell_ratio)  
**최대 원점수**: 65점 → 0~100 정규화

### 가점 항목

| 조건 | 원점수 | 설명 |
|---|---|---|
| 외국인 매수 전환 (1~2일, 직전 매도 후) | +20 | 새 수급 유입 — 최고 신호 |
| 기관 매수 전환 (1~2일, 직전 매도 후) | +20 | 새 수급 유입 |
| 외국인 3~5일 연속 매수 | +15 | 지속 매집 |
| 기관 3~5일 연속 매수 | +15 | 지속 매집 |
| 외국인+기관 동반 매수 (당일) | +10 | 스마트머니 동시 유입 |
| 5일 누적 외국인+기관 동반 순매수 | +10 | 누적 매집 확인 |
| 공매도 비율 1% 미만 | +5 | 하방 압력 낮음 |

### 감점 항목

| 조건 | 원점수 | 설명 |
|---|---|---|
| 외국인 3일+ 연속 매도 | -15 | 수급 이탈 |
| 기관 3일+ 연속 매도 | -15 | 수급 이탈 |

## 축 3. 가치 매력도 (Valuation Attractiveness) — 0~100

**원칙**: 목표주가 괴리율이 1순위. PBR/ROE는 보조.  
**데이터**: `stock_cache` (target_price, forward_per, per, pbr, roe, dividend_yield, invest_opinion)  
**최대 원점수**: 80점 → 0~100 정규화

### 가점 항목

| 조건 | 원점수 | 설명 |
|---|---|---|
| 목표주가 대비 현재가 괴리 30%+ | +35 | 강한 저평가 신호 |
| 목표주가 대비 현재가 괴리 20%+ | +25 | 유의미한 상승여력 |
| 목표주가 대비 현재가 괴리 10%+ | +15 | 소폭 상승여력 |
| Forward PER < Trailing PER (이익 성장) | +20 | 이익 증가 예상 |
| PBR < 1.0 + ROE > 10% (저평가 우량주) | +20 | 가치+수익성 복합 |
| PBR < 0.7 (순자산 대비 극저평가) | +15 | 자산가치 매력 |
| 배당수익률 3%+ | +10 | 하방 지지선 |
| 애널리스트 Strong Buy 컨센서스 (invest_opinion ≥ 4) | +10 | 전문가 동의 |

### 감점 항목

| 조건 | 원점수 | 설명 |
|---|---|---|
| PER > 50 (성장주 제외 고평가) | -10 | 밸류 부담 |
| 목표주가 < 현재가 (하향 조정) | -10 | 전문가 하향 의견 |

## 축 4. SMS 신호 보너스 (Signal Bonus) — 0~100

**원칙**: 신호 없어도 높은 점수 가능. 신호는 10% 가중치 내에서만 작동.

| 조건 | 점수 |
|---|---|
| 오늘 2개+ 소스 신호 | 60 |
| 오늘 1개 소스 신호 | 40 |
| 3~10일 내 신호 + 현재가 ≤ 신호가 | 50 |
| 3~10일 내 신호 + 현재가 > 신호가 | 30 |
| 30일 내 3회+ 반복 신호 | 20 |
| 신호 없음 | 0 |

## 리스크 감점 (Risk Penalty)

리스크 점수는 종합점수에서 최대 20% 감산.

| 조건 | 페널티 |
|---|---|
| 관리종목 | 20% |
| 전환사채(CBW) 최근 발행 | 10% |
| 최근 감사의견 비적정 | 15% |
| RSI 75+ + 5일 +20%+ 동시 | 10% |

## 스코어링 실행 아키텍처

```
크론 (장 마감 후, 1일 1회)
  ├── stock_cache 전체 조회 (4,242종목)
  ├── daily_prices 최근 65일 배치 조회
  ├── calcScore() 4,242종목 × 4축 계산
  └── stock_ranking_snapshot 저장

프론트엔드
  ├── 스냅샷 로드 (실시간 계산 없음)
  ├── live 가격 오버레이 (usePriceRefresh)
  └── 클라이언트 필터/정렬
```

## 검증 시뮬레이터

### 목적

점수 체계가 실제 수익률을 예측하는지 정량적으로 검증.

### 기능 명세

**1. 날짜별 스코어 재현**
- 특정 과거 날짜를 선택하면, 해당 시점의 `daily_prices` + `stock_cache`를 기반으로 새 점수 체계로 재계산
- "그 날 삼성전자 점수는 몇 점이었을까" 확인 가능

**2. 수익률 추적**
- 점수 계산 기준일로부터 1주/2주/4주/8주 후 실제 수익률 측정
- 점수 구간별(A+/A/B+/B/C/D) 평균 수익률 통계 산출

**3. 점수-수익률 상관관계 차트**
- X축: 예측 점수, Y축: 실제 N주 후 수익률
- 유효한 시스템이면 우상향 상관관계

**4. 종목별 히스토리**
- 특정 종목(삼성전자 등) 선택 → 날짜별 점수 변화 + 실제 주가 오버레이
- "언제 A등급이었고, 그 후 실제로 올랐는가" 시각화

**5. 백테스트 요약**
- 기간 선택 → A+ 종목만 매수 시 평균 수익률 vs 시장 평균
- 등급별 승률(양수 수익률 비율)

### 구현 방식

- **경로**: `/signals?tab=backtest` 또는 별도 `/tools/score-simulator`
- **데이터**: `daily_prices` 테이블 (이미 있음) + `signals` 테이블
- **API**: `/api/v1/score-simulator?date=YYYY-MM-DD&symbol=005930`
- **UI**: 날짜 피커 + 종목 검색 + 차트 (Recharts 활용)

2026-04-01 기준 시뮬레이션:

| 축 | 조건 | 원점수 |
|---|---|---|
| 기술전환 | 52주 저점 +12% 반등 + RSI 42 | 40/65 |
| 수급 | 외국인 3일 연속 매수 + 기관 전환 | 45/65 |
| 가치 | 목표주가 괴리 25% + Forward PER 성장 + PBR 1.0 | 55/80 |
| 신호 | 없음 | 0 |

정규화 후 가중 합산 (대형주):
```
(62 × 30 + 69 × 25 + 69 × 35 + 0 × 10) / 100 = 62점 → B+
```

현행 시스템: D → 새 시스템: **B+**
