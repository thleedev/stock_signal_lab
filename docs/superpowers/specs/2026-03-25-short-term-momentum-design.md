# 초단기 반응형 종목 점수 모델 설계

> 작성일: 2026-03-25

## 1. 개요

### 목적

기존 AI 종합 점수 체계와 **별도로**, "오늘 사면 내일~모레 반응이 올 자리인가"를 정량 평가하는 초단기 모멘텀 순위 모델.

### 철학 (30년차 애널리스트 3대 원칙)

1. **거래량은 가격에 선행한다** — 가격과 거래량은 하나의 신호. 거래량 없는 상승은 허상이고, 거래량 동반 상승만이 내일의 연속성을 보장한다.
2. **스마트머니를 따라가라** — 외국인/기관이 "오늘" 산 종목이 "내일" 더 간다. 5일 누적보다 당일+2일 흐름이 초단기 핵심.
3. **늦은 자리는 손실의 시작이다** — 좋은 종목을 찾는 것보다 나쁜 타이밍을 거르는 것이 수익률을 결정한다.

### 기존 모델과의 차이

| | 기존 종합 모델 | 초단기 모델 |
|---|---|---|
| **핵심 질문** | "좋은 종목인가?" | "내일 더 올 자리인가?" |
| **가격×거래량** | 별도 축 | **통합 매트릭스** |
| **종가 위치** | 없음 | 핵심 지표 |
| **거래대금** | 없음 | 절대+상대 이중 평가 |
| **밸류에이션** | PER/PBR/ROE 10% | 5% (극저평가 보너스) |
| **52주 위치** | 핵심 40점 | 없음 |
| **신호 평가** | 30일 반복성 | 신선도 (오늘>어제>3일) |
| **수급 시계** | 5일 누적 | 당일+2일 |
| **프로그램 매매** | 없음 | 반영 (v1: graceful degradation) |
| **섹터** | 상대강도 | 테마 상승 순위 |
| **리스크 관리** | 없음 | 15% 감산 |
| **1차 필터** | 없음 | 6개 조건 사전 제거 |

## 2. 점수 공식

### 기본 공식

```
기본 점수 = 모멘텀_norm × W1/100 + 수급_norm × W2/100 + 촉매_norm × W3/100 + 밸류_norm × W4/100
최종 점수 = 기본 점수 - 리스크_norm × R/100
           → clamp(0, 100)

기본 가중치 (W1+W2+W3+W4 = 100):
  W1(모멘텀)=45, W2(수급)=28, W3(촉매)=22, W4(밸류)=5

리스크 가중치 (별도):
  R=15
```

### 정규화 규칙

각 카테고리의 원점수(raw)는 범위가 서로 다르므로, **0~100으로 정규화** 후 가중치를 적용한다.

```
정규화 공식: norm = clamp((raw - MIN) / (MAX - MIN) × 100, 0, 100)
```

| 카테고리 | 원점수 범위 (MIN~MAX) | 정규화 공식 | 비고 |
|----------|----------------------|-------------|------|
| 모멘텀 | -10 ~ 90 | `(raw + 10) / 100 × 100` | A+B+C+D 합산 |
| 수급 | -25 ~ 55 | `(raw + 25) / 80 × 100` | A+B+C 합산 |
| 촉매 | -10 ~ 60 | `(raw + 10) / 70 × 100` | A+B+C 합산 |
| 밸류 | 0 ~ 75 | `raw / 75 × 100` | Forward/폴백 + ROE |
| 리스크 | 0 ~ 100 | `clamp(|raw|, 0, 100)` | 감점 절대값 합산, 상한 100 |

### 가중치 요약

| 카테고리 | 기본 가중치 | 역할 |
|----------|------------|------|
| 모멘텀 (가격×거래량) | 45% | 내일 연속 상승 가능성 |
| 수급 | 28% | 스마트머니 방향 |
| 촉매 (신호/테마) | 22% | 내일 더 갈 이유 |
| 밸류에이션 | 5% | 저평가 + 촉매 = 상승폭 확대 |
| 리스크 패널티 | 15% (감산) | 늦은 자리 회피 |

가중치는 사용자 커스터마이징 가능 (UI 슬라이더, localStorage 저장).

## 3. 대상 종목 및 1차 필터

### 대상

오늘 BUY 신호가 있는 종목 (기존 AI 추천과 동일한 후보군).

### 1차 필터 (점수 계산 전 후보 제거)

| 조건 | 기준 | 근거 |
|------|------|------|
| 당일 등락률 | ≥ +0.5% AND ≤ +8% | 안 움직이면 촉매 없음, 급등이면 늦음 |
| 거래대금 | ≥ 200억 | 진입/탈출 불가능한 종목은 추천 불가 |
| 종가 위치 | ≥ 0.5 (고가=저가 시 1.0 간주) | 매수세가 장 마감까지 유지 |
| 수급 | 외국인/기관 중 1개 이상 순매수 | 개인만의 상승은 지속성 없음 (v1: 프로그램 제외) |
| 과열 | 3일 누적 +20% 초과 제외 | 고점 추격 원천 차단 |
| 신호 | 최근 3일 내 BUY 또는 당일 섹터 강세 | 촉매 최소 조건 |

> **구간 표기 규칙**: 모든 조건은 **이상/미만** 사용 — 예: `≥ 0.5% AND < 8%`. 경계값은 낮은 구간에 포함.

## 4. 카테고리별 점수 상세

> **구간 경계 규칙**: 모든 구간은 `[이상, 미만)` 형식. 예: `[+1%, +3%)`는 +1% 이상 +3% 미만. 예외적으로 최하위/최상위 구간은 열린 구간.

### 4.1 모멘텀 점수 (기본 가중치 45%)

가격과 거래량을 하나의 축으로 통합. "주가가 올랐는데 거래량도 터졌나?"

**원점수 범위: -10 ~ 90** → 정규화: `(raw + 10) / 100 × 100`

#### A. 가격-거래량 동반 시그널 (핵심, 최대 35점)

거래량 배수 = 당일 거래량 / 20일 평균 거래량

| | 거래량 ≥ 2배 | 거래량 [1.5배, 2배) | 거래량 < 1.5배 |
|---|---|---|---|
| **등락률 [+1%, +3%)** | **+35** (최적) | +28 | +18 |
| **등락률 [+3%, +6%)** | +30 | +25 | +15 |
| **등락률 [+0.5%, +1%)** | +22 | +15 | +8 |
| **등락률 [+6%, +8%)** | +15 | +12 | +5 |
| **등락률 [-1%, +0.5%)** | +12 | +8 | +3 |
| **등락률 ≥ +8%** | -5 | -8 | -10 |

> 1차 필터 통과 종목의 등락률 범위(+0.5%~+8%)와 일치하도록 매트릭스 구간을 조정.
> "+1~3% 상승 + 거래량 2배 폭발 = 막 시동 걸렸고, 돈이 몰려드는 중" — 내일 가장 갈 확률 높은 조합.

#### B. 종가 위치 (최대 20점)

```
종가위치 = (종가 - 저가) / (고가 - 저가)
          고가 == 저가인 경우 → 1.0 (상한가 등 가격 고정)
          open이 null인 경우 → 이 항목 0점 처리
```

| 종가 위치 | 점수 | 해석 |
|-----------|------|------|
| ≥ 0.8 | +20 | 양봉 장악형 — 매수세 완전 장악 |
| [0.6, 0.8) | +12 | 상체 양봉 — 매수 우위 |
| [0.4, 0.6) | +3 | 팽이 — 방향 미정 |
| < 0.4 | -10 | 윗꼬리 음봉 — 매도 압력 우세 |

#### C. 갭업 + 전일 패턴 (최대 20점, **합산 후 clamp**)

```
갭 = (당일 시가 - 전일 종가) / 전일 종가
    시가(open)가 null인 경우 → 이 항목 0점 처리
```

해당 조건이 여러 개 충족될 경우 **합산 후 20점 clamp**.

| 조건 | 점수 | 근거 |
|------|------|------|
| 전일 장대양봉(몸통 ≥ 3%) + 당일 소폭 갭업([+1%, +3%)) | +20 | 전일 에너지 축적 + 당일 확인 |
| 전일 장대양봉 + 당일 보합 출발 | +15 | 에너지 축적, 아직 출발 전 |
| 당일 갭업 [+1%, +3%) (전일 무관) | +12 | 장 시작부터 매수세 |
| 2일 연속 양봉 | +10 | 단기 추세 확인 |
| 전일 고점 돌파 | +10 | 저항선 돌파 |
| 3일 박스 상단 돌파 | +8 | 레인지 탈출 |
| 갭업 ≥ +3% | +3 | 되돌림 위험 |
| 갭다운 (갭 < 0%) | 0 | |

> **적용 방식**: 갭업 조건(상위 3개) 중 하나만 선택 + 패턴 조건(하위 4개)은 복수 합산 가능 → 전체 합산 후 20점 clamp.

#### D. 거래대금 (최대 15점)

거래량 배수와 별개로, 절대 거래대금은 유동성과 신뢰도의 지표.

| 거래대금 | 점수 |
|----------|------|
| ≥ 1,000억 | +15 |
| [500억, 1,000억) | +10 |
| [200억, 500억) | +5 |

### 4.2 수급 점수 (기본 가중치 28%)

"누가 사고 있나?" — 초단기에서 스마트머니 방향은 가격보다 정직하다.

**원점수 범위: -25 ~ 55** → 정규화: `(raw + 25) / 80 × 100`

> **v1 프로그램 매매 미지원**: 프로그램 매매 데이터 미확보 시 해당 항목(A: +8점, B: +5점)은 0점 처리. 수급 원점수의 실질 범위는 -25 ~ 42가 되며, 정규화 공식은 동일 적용 (자연스럽게 만점이 낮아짐). v2에서 프로그램 데이터 확보 시 원래 범위 복원.

#### A. 당일 주체별 매수 (최대 30점, v1: 최대 22점)

| 조건 | 점수 | 근거 |
|------|------|------|
| 외국인 당일 순매수 | +10 | 정보 우위 |
| 기관 당일 순매수 | +10 | 리서치 기반 |
| 프로그램 순매수 | +8 (v1: 0) | 알고리즘 모멘텀 추종 |
| 외국인+기관 동반 순매수 | +12 보너스 | 동반 매수는 가장 강한 수급 신호 |

#### B. 2일 연속성 (최대 15점, v1: 최대 10점)

| 조건 | 점수 |
|------|------|
| 외국인 2일 연속 순매수 | +5 |
| 기관 2일 연속 순매수 | +5 |
| 프로그램 2일 연속 순매수 | +5 (v1: 0) |

#### C. 수급 경고 (감점)

| 조건 | 점수 |
|------|------|
| 외국인/기관 둘 다 순매도 | -15 |
| 3일 이상 연속 매도 (외국인 or 기관) | -10 |

### 4.3 촉매 점수 (기본 가중치 22%)

"왜 내일 더 갈 수 있나?" — 모멘텀과 수급이 현재를 말해주면, 촉매는 내일을 말해준다.

**원점수 범위: -10 ~ 60** → 정규화: `(raw + 10) / 70 × 100`

#### A. 신호 신선도 (최대 25점)

| 조건 | 점수 |
|------|------|
| 오늘 신규 BUY (3개 소스 동시) | +25 |
| 오늘 신규 BUY (2개 소스) | +20 |
| 오늘 신규 BUY (1개 소스) | +15 |
| 어제 BUY 발생 | +10 |
| 최근 3일 내 첫 신호 | +5 |
| 5일 이상 지난 신호 | 0 |

#### B. 섹터/테마 모멘텀 (최대 25점)

섹터 등락률 집계: `stock_cache.price_change_pct`를 `stock_info.sector` 기준으로 GROUP BY → 섹터 평균 등락률 및 순위 산출.

| 조건 | 점수 | 근거 |
|------|------|------|
| 당일 해당 섹터 상승률 상위 3 | +20 | 핫 섹터 소속 |
| 섹터 내 상승률 상위 30% | +15 | 섹터 주도주 |
| 섹터 평균 이상 상승 | +8 | 섹터 동행 |
| 섹터 약한데 종목만 단독 상승 | +3 | 개별 재료 가능성 |
| 섹터 전체 약세 | -10 | 역풍 |

#### C. 신호가 대비 현재 위치 (최대 10점)

| 조건 | 점수 |
|------|------|
| 현재가 ≤ 신호가 | +10 |
| 신호가 대비 [0%, +3%) | +5 |
| 신호가 대비 [+3%, +7%) | 0 |
| 신호가 대비 ≥ +7% | 리스크로 이관 |

### 4.4 밸류에이션 점수 (기본 가중치 5%)

초단기에서 밸류에이션은 주역이 아니지만, 극단적 저평가 종목이 촉매를 만났을 때 상승 폭이 커지는 효과를 반영.

**원점수 범위: 0 ~ 75** → 정규화: `raw / 75 × 100`

#### Forward 데이터 있는 경우

| 지표 | 조건 | 점수 |
|------|------|------|
| **Forward PER** | < 8 | +30 |
| | [8, 12) | +20 |
| | [12, 20) | +10 |
| | ≥ 20 또는 적자 | 0 |
| **목표주가 괴리율** | ≥ 30% | +25 |
| | [15%, 30%) | +15 |
| | [5%, 15%) | +5 |
| | < 5% 또는 없음 | 0 |

#### Forward 데이터 없는 경우 (폴백)

| 지표 | 조건 | 점수 |
|------|------|------|
| **PBR** | < 0.5 | +30 |
| | [0.5, 1.0) | +15 |
| | [1.0, 1.5) | +5 |

#### 공통

| 지표 | 조건 | 점수 |
|------|------|------|
| **ROE** | > 15% | +20 |
| | (10%, 15%] | +10 |
| | (5%, 10%] | +5 |

→ 별도 `calcShortTermValuationScore()` 함수로 구현 (기존 `calcValuationScore`의 구조를 참조하되 배점 스케일이 다르므로 새 함수 필요).

### 4.5 리스크 패널티 (기본 가중치 15%, 감산)

> "한 번의 큰 손실이 열 번의 수익을 날린다."

**원점수 범위: 0 ~ 100** → 감점 절대값 합산 후 clamp(0, 100)

#### A. 과열 패널티

| 조건 | 감점 | 근거 |
|------|------|------|
| 당일 ≥ +12% 급등 | -20 | 내일 차익실현 쏟아짐 |
| 3거래일 누적 ≥ +20% | -20 | 단기 과열 극심 |
| 당일 ≥ +10% + 거래량 < 20일 평균 | -15 | 허매수 — 가장 위험한 패턴 |

#### B. 캔들 위험 신호

| 조건 | 감점 | 근거 |
|------|------|------|
| 윗꼬리 ≥ 몸통 2배 (open이 null이면 skip) | -12 | 고점 매도 압력 |
| 종가 < 시가 (음봉 전환) + 등락률 양전 (open null이면 skip) | -10 | 장중 반전 — 매수세 이탈 |
| 장 마감 급락 (종가 밀림: 종가 < 시가 × 0.97) | -12 | 기관/프로그램 마감 매도 |

#### C. 추격매수 위험

| 조건 | 감점 | 근거 |
|------|------|------|
| 신호가 대비 현재가 [+7%, +12%) | -12 | 추격 구간 |
| 신호가 대비 현재가 ≥ +12% | -20 | 고점 물림 위험 |
| 거래대금 < 100억 + 등락률 > +3% | -10 | 유동성 트랩 |
| 2일 연속 장대양봉(각 ≥ +5%) 후 3일째 | -15 | 3일째 고점 확률 높음 |

## 6. 점수 산출 예시

### 예시 1: B+ 등급 — "막 시동 걸린 이상적 종목"

```
종목: ABC전자 (섹터: 반도체)
당일: +2.1%, 거래량 20일 평균 대비 2.8배, 거래대금 850억
종가위치: 0.85 (양봉 장악형)
전일: +3.5% 장대양봉
갭업: +1.2%
수급: 외국인 순매수 +15만주, 기관 순매수 +8만주 (프로그램 v1 미지원)
신호: 오늘 BUY 2개 소스 동시
섹터: 반도체 당일 상승률 2위
Forward PER: 9.5, 목표주가 괴리: +25%, ROE: 12%
```

**원점수 산출:**

| 카테고리 | 항목별 | 원점수 |
|----------|--------|--------|
| 모멘텀 | 가격×거래량(+35) + 종가위치(+20) + 전일양봉+갭업 clamp(+20) + 거래대금 850억(+10) | **85** |
| 수급 | 외국인(+10) + 기관(+10) + 동반(+12) = 32 (v1: 프로그램 0) | **32** |
| 촉매 | 신호 2소스(+20) + 섹터 상위3(+20) + 신호가 이내(+5) | **45** |
| 밸류 | Forward PER 9.5<12(+20) + 목표주가 25%≥15%(+15) + ROE 12%>10%(+10) | **45** |
| 리스크 | 해당 패널티 없음 | **0** |

**정규화:**

| 카테고리 | 공식 | 정규화 값 |
|----------|------|-----------|
| 모멘텀 | (85+10)/100×100 | **95.0** |
| 수급 | (32+25)/80×100 | **71.3** |
| 촉매 | (45+10)/70×100 | **78.6** |
| 밸류 | 45/75×100 | **60.0** |
| 리스크 | 0 | **0** |

**최종 점수:**

```
= 95.0×0.45 + 71.3×0.28 + 78.6×0.22 + 60.0×0.05 - 0×0.15
= 42.75 + 19.96 + 17.29 + 3.00 - 0
= 83.0 → B+
```

### 예시 2: D등급 — "이미 늦은 종목"

```
종목: XYZ테크
당일: +11.5%, 거래량 평이, 거래대금 180억
종가위치: 0.35 (윗꼬리)
3일 누적: +18%
수급: 외국인 매도, 기관 매도
신호: 5일 전 BUY
```

→ **1차 필터에서 탈락**: 등락률 +8% 초과, 거래대금 200억 미달, 종가위치 0.5 미만, 수급 없음

### 예시 3: 리스크 패널티 작동 예시

```
종목: DEF바이오
당일: +7.5%, 거래량 2.5배, 거래대금 450억
종가위치: 0.72
수급: 외국인 순매수, 기관 매도
3일 누적: +22%
신호: 오늘 BUY 1소스, 신호가 대비 +9%
```

**원점수:**
- 모멘텀: 가격×거래량(+12) + 종가위치(+12) + 거래대금(+10) = **34**
- 수급: 외국인(+10) = **10**
- 촉매: 신호 1소스(+15) + 신호가 +9% → 리스크 이관(0) = **15**
- 밸류: (생략) = **20**
- 리스크: 3일 누적 +22%(-20) + 신호가 대비 +9%(-12) = **|-32| → clamp 32**

**최종:**
```
정규화: 모멘텀 44.0, 수급 43.8, 촉매 35.7, 밸류 26.7, 리스크 32.0
= 44.0×0.45 + 43.8×0.28 + 35.7×0.22 + 26.7×0.05 - 32.0×0.15
= 19.8 + 12.3 + 7.9 + 1.3 - 4.8
= 36.5 → D
```

> 모멘텀과 수급은 나쁘지 않지만, 3일 누적 과열 + 신호가 추격 패널티로 D등급 추락.

## 7. 기술 아키텍처

### 접근법

기존 `ai-recommendation/` 모듈 구조를 확장. 별도 스코어링 함수를 추가하고, DB는 `ai_recommendations` 테이블에 `model_type` 컬럼과 `score_breakdown` JSONB 컬럼으로 구분.

### 파일 구조

```
web/src/lib/ai-recommendation/
  ├── index.ts                      (기존 — generateRecommendations)
  ├── short-term-momentum.ts        (신규 — generateShortTermRecommendations)
  ├── short-term/
  │   ├── momentum-score.ts         (가격×거래량 매트릭스 + 종가위치 + 갭업/패턴 + 거래대금)
  │   ├── supply-score.ts           (당일 주체별 + 2일 연속 + 경고)
  │   ├── catalyst-score.ts         (신호 신선도 + 섹터/테마 + 신호가 위치)
  │   ├── valuation-score.ts        (신규 — calcShortTermValuationScore)
  │   ├── risk-penalty.ts           (과열 + 캔들 위험 + 추격)
  │   └── pre-filter.ts             (1차 필터 6개 조건)
  ├── signal-score.ts               (기존)
  ├── technical-score.ts            (기존)
  ├── valuation-score.ts            (기존 — 종합 모델용)
  └── supply-score.ts               (기존 — 종합 모델용)
```

### API 엔드포인트

```
GET  /api/v1/ai-recommendations?model=short_term
POST /api/v1/ai-recommendations/generate  (body: { model: 'short_term', weights?: {...} })
```

기존 엔드포인트에 `model` 파라미터 추가. 기본값 `'standard'`로 하위 호환 유지.

**기존 GET 라우트 수정**: `WHERE model_type = 'standard'` 기본 필터 추가하여 초단기 모델 결과가 기존 종목추천 탭에 노출되지 않도록 보장.

### DB 변경

```sql
-- ai_recommendations 테이블에 컬럼 추가
ALTER TABLE ai_recommendations ADD COLUMN model_type TEXT DEFAULT 'standard';
ALTER TABLE ai_recommendations ADD COLUMN score_breakdown JSONB;

-- 인덱스
CREATE INDEX idx_ai_rec_model_date ON ai_recommendations(model_type, date);
```

- `model_type`: `'standard'` | `'short_term'`
- `score_breakdown`: 모델별 상세 점수를 JSON으로 저장 (기존 컬럼 `signal_score`, `technical_score` 등은 standard 모델 전용으로 유지)
- **DELETE 패턴 수정**: `DELETE FROM ai_recommendations WHERE date = $1 AND model_type = $2` — model_type별 분리 삭제로 race condition 방지

### 타입 정의

```typescript
// types/ai-recommendation.ts 확장

export type ModelType = 'standard' | 'short_term';

export interface ShortTermWeights {
  momentum: number;   // 기본 45
  supply: number;     // 기본 28
  catalyst: number;   // 기본 22
  valuation: number;  // 기본 5
  risk: number;       // 기본 15 (감산)

  momentum: 45,
  supply: 28,

};

  momentum: number;      // 정규화 후 0~100
  supply: number;        // 정규화 후 0~100
  catalyst: number;      // 정규화 후 0~100
  valuation: number;     // 정규화 후 0~100
  risk: number;          // 정규화 후 0~100
```

### 데이터 소스

| 데이터 | 소스 | 비고 |
|--------|------|------|
| 당일 OHLCV | `daily_prices` (장 마감 후) 또는 네이버 API 실시간 조회 (장중) | 종가위치, 갭, 거래량 |
| 전일/3일 OHLCV | `daily_prices` | 패턴 분석 |
| 20일 평균 거래량/거래대금 | `daily_prices` 집계 | 상대 거래량 |
| 외국인/기관 수급 | `stock_cache` (foreign_net_qty, institution_net_qty, streak) | 당일+streak |
| 신호 | `signals` 테이블 | 신선도 계산 (signal_time 기준 몇 일 전인지) |
| 섹터 정보 | `stock_info.sector` + `stock_cache.price_change_pct` JOIN → GROUP BY sector | 섹터 평균 등락률 및 순위 |
| 밸류에이션 | `stock_cache` (forward_per, target_price, invest_opinion, per, pbr, roe) | |
| 거래대금 | `daily_prices.volume × daily_prices.close` 또는 별도 trading_value 필드 | 20일 평균 대비 |

> **장중 vs 장 마감**: 이 모델은 **장 마감 후 데이터 기준**으로 설계됨. 장중에 탭 진입 시에는 최신 `daily_prices` + `stock_cache` 데이터로 계산하되, 당일 OHLCV가 불완전할 수 있음을 UI에 "(장중 추정)" 라벨로 표시.

### 갱신 전략

- 탭 진입 시 자동 생성/갱신
- `generated_at` 기준 5분 경과 시 재진입하면 자동 재생성 (stale-while-revalidate)
- 수동 "재계산" 버튼으로 즉시 재생성 가능

### 에러 핸들링

기존 코드의 `Promise.allSettled` 패턴을 따름:
- 외부 API 실패 시 해당 종목의 누락 데이터는 0점 처리
- Supabase 쿼리 타임아웃 시 캐시된 이전 결과 반환
- 부분 실패 시에도 가능한 종목만으로 순위 생성

## 9. 프로그램 매매 데이터

### 현재 상태

기존 `stock_cache`에 프로그램 매매 필드가 없음. 네이버 증권 API에서 프로그램 순매수 데이터 수집 필요.

### 방안

1. `stock_cache`에 `program_net_qty`, `program_streak` 필드 추가
2. `signal-data-enricher.ts`에서 프로그램 매매 데이터 추가 수집
3. **v1**: 데이터 소스 미확보 시 프로그램 매매 항목은 0점 처리 (graceful degradation)
4. **v2**: 프로그램 데이터 확보 후 수급 점수에 정상 반영

## 10. 향후 확장 (v2+)

- **프로그램 매매 데이터**: 네이버 증권 API 연동 → 수급 점수 정상화
- **재료/뉴스 점수**: 뉴스 API 연동 시 촉매 카테고리에 재료 항목 추가
- **VI/과열 종목 감지**: 한국거래소 VI 발동 데이터 연동
- **백테스트**: 과거 데이터로 모델 성능 검증 및 가중치 최적화
- **장중 실시간 재계산**: 현재 30초 폴링 데이터로 점수 실시간 보정
