# 메인 대시보드 재설계 설계 문서

**날짜:** 2026-03-16
**대상 페이지:** `/` (메인 대시보드)
**목표:** 각 페이지의 핵심 요약을 그리드 위젯으로 표시하는 허브형 대시보드로 재편

---

## 1. 배경 및 목표

현재 메인 대시보드는 앱 기능이 확장되면서 추가된 페이지들(위험 경보 시스템, 가상 포트폴리오, 소스별 포트폴리오)을 반영하지 못하고 있다. 목표는 모든 주요 페이지의 핵심 지표를 요약 카드(위젯) 형태로 배치하여, 사용자가 대시보드 하나에서 전체 현황을 파악하고 각 페이지로 이동할 수 있는 허브로 재편하는 것이다.

---

## 2. 레이아웃 구조

```
┌─────────────────────────────────────────────────┐
│  위험 경보 배너 (전체 너비) → /market            │
└─────────────────────────────────────────────────┘

┌─────────┬─────────┬─────────┬─────────────────┐
│ 라씨매매 │ 스톡봇   │ 퀀트    │  투자 시황      │
│ → /signals         │         │  → /market      │
└─────────┴─────────┴─────────┴─────────────────┘

┌──────────────────────┬──────────────┬──────────┐
│  관심종목             │  투자 현황   │ 가상 PF  │
│  (col-span-2)        │  → /investment│→/my-pf  │
│  → /stocks           │              │          │
└──────────────────────┴──────────────┴──────────┘

┌─────────────┬─────────────┬─────────────────────┐
│ 라씨 포트폴리오│스톡봇 포트폴리오│퀀트 포트폴리오  │
│→/portfolio/lassi│→/portfolio/stockbot│→/portfolio/quant│
└─────────────┴─────────────┴─────────────────────┘
```

## 3. 위젯별 상세 명세

### 3.1 위험 경보 배너 (전체 너비)
- **데이터 소스:** `market_score_history` 최신 레코드 → `risk_index` 필드
- **표시 내용:**
  - 위험 지수 숫자 (0~100)
  - 위험 레벨 라벨: 안전(0~25) / 주의(26~50) / 위험(51~75) / 극위험(76~100)
  - 위험 지표 개수 (예: "4/12개 위험")
- **인터랙션:** 카드 전체 클릭 → `/market`
- **구현:** 기존 `RiskAlertBanner` 컴포넌트 재사용 또는 경량화된 버전 추출

### 3.2 신호 카드 × 3 (라씨매매 / 스톡봇 / 퀀트)
- **데이터 소스:** `signals` 테이블, 오늘 날짜 기준
- **표시 내용:**
  - 소스명 + 색상 테마 (라씨=빨강, 스톡봇=초록, 퀀트=파랑)
  - BUY 신호 수 / SELL 신호 수
- **인터랙션:** 클릭 → `/signals?source={lassi|stockbot|quant}`

### 3.3 투자 시황 카드
- **데이터 소스:** `market_score_history` 최신 레코드
- **표시 내용:**
  - 마켓 스코어 (숫자)
  - 이벤트 리스크 스코어 (숫자)
  - 가장 가까운 이벤트 1개 (`market_events` 테이블)
- **인터랙션:** 클릭 → `/market`

### 3.4 관심종목 카드 (2열 폭)
- **데이터 소스:** `stock_cache` (favorites=true), 최대 5개
- **표시 내용:**
  - 종목명 + 현재가 + 일일 등락률 (색상: 상승=빨강, 하락=파랑)
- **인터랙션:** 카드 헤더 클릭 → `/stocks`

### 3.5 투자 현황 카드
- **데이터 소스:** `watchlist` 테이블
- **표시 내용:**
  - 보유 종목 수
  - 평균 수익률 (%)
- **인터랙션:** 클릭 → `/investment`

### 3.6 가상 포트폴리오 카드
- **데이터 소스:** `virtual_trades` / `portfolio_snapshots` 테이블
- **표시 내용:**
  - 전체 가상 PF 누적 수익률
  - 3개 소스별 간략 성과 (소스명 + 수익률)
- **인터랙션:** 클릭 → `/my-portfolio`

### 3.7 소스별 포트폴리오 카드 × 3
- **데이터 소스:** `portfolio_snapshots` 또는 `virtual_trades`, 소스별 집계
- **표시 내용:**
  - 소스명 (라씨매매 / 스톡봇 / 퀀트)
  - 해당 소스 포트폴리오 수익률
  - 보유 종목 수
- **인터랙션:** 클릭 → `/portfolio/{lassi|stockbot|quant}`

## 4. 데이터 페칭 전략

- **서버 컴포넌트 (`page.tsx`):** 초기 렌더링 시 모든 위젯 데이터를 서버에서 일괄 페칭 (Supabase 서버 클라이언트 사용)
- **실시간 갱신:** 관심종목 가격만 클라이언트에서 주기적 폴링 (`usePriceRefresh` 훅 유지)
- **위험 지수:** 서버에서 최신값 1개만 조회 (기존 방식 유지)

---

## 5. 컴포넌트 구조

```
app/page.tsx (서버 컴포넌트)
  ├── RiskAlertBannerWidget      # 위험 경보 배너 (경량화)
  ├── SignalSummaryCard × 3      # 신호 요약 카드
  ├── MarketSummaryCard          # 투자 시황 요약 카드
  ├── WatchlistWidget            # 관심종목 위젯
  ├── InvestmentSummaryCard      # 투자 현황 요약 카드
  ├── VirtualPortfolioCard       # 가상 PF 요약 카드
  └── SourcePortfolioCard × 3    # 소스별 포트폴리오 카드
```

기존 `DashboardPrices` 컴포넌트는 `WatchlistWidget`으로 대체하며, 관심종목 실시간 가격 갱신 로직은 유지한다.

## 6. 영향 범위

- **수정:** `web/src/app/page.tsx` (전면 재작성)
- **신규:** `web/src/components/dashboard/` 하위 위젯 컴포넌트들
- **재사용:** `RiskAlertBanner`, `usePriceRefresh`, Supabase 클라이언트
- **제거:** 기존 `EventSummaryCard` 인라인 배치 → `MarketSummaryCard`로 대체
- **변경 없음:** 각 개별 페이지 (`/market`, `/signals`, `/investment` 등)
